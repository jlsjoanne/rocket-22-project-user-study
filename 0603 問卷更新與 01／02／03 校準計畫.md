# 0603 問卷更新與 01/02/03 校準計畫

## Summary

- 將新版問卷 `跨世代／跨人生階段交流平台需求調查v2 (回覆)-0603.xlsx` 納入正式基準。
- 有效回覆數由 55 更新為 65，開放題非空回覆由 26 更新為 30。
- 不從零重做研究流程；採用「更新 01、影響檢查 02、校準 03」方式。
- 「一對一匿名經驗交流信箱」定位為團隊 MVP 的**主要產品假設**，不是問卷已直接證明的唯一最佳方案。

## Key Changes

- 覆寫更新現有 `01_問卷重點整理.md`：
  - 更新樣本數、年齡分布、主要比例、開放題數量。
  - 重新計算 `F-xx` 數據。
  - 保留探索性語氣，避免寫成市場定論。
  - 加入判斷：資料支持匿名、相關經驗、安全、非即時互動；一對一信箱是團隊選擇的 MVP 承載假設。

- 更新 `02_需求洞察整理.md`：
  - 維持 5 個正式洞察，不新增第 6 洞察。
  - 強化 `IN-01 安全可信的交流環境` 為最高優先，因新版資料中安全審核/檢舉已成為提高使用意願的最高因素之一。
  - 將「平台差異化」與「分享者回覆誘因/供需匹配」加入待驗證問題。
  - 補明：一對一信箱可行性取決於主題/經驗配對、非即時互動、平台內安全邊界與清楚差異化。

- 補 Affinity Diagram 轉換建議：
  - 以 `F-xx`、`T-xx`、`IN-xx` 轉為 FigJam 便利貼。
  - 先用 5 個既有洞察作為分群：安全可信、相關經驗、低壓力非即時、導覽輔助、清楚新手流程。
  - 每群補上洞察句、證據來源、信心等級與待驗證問題。
  - 02 可視為文字版正式洞察，但仍建議補 FigJam/Affinity 作為團隊共識證據。

- 暫停並校準 `03_假設型 Persona初稿.md`：
  - 不直接進 User Stories。
  - Persona 仍維持 `經驗尋求者`、`經驗分享者` 兩類。
  - 加入一對一信箱作為主要產品假設下的使用情境與待驗證假設。
  - 強化分享者的回覆誘因、拒絕/略過、界線設定與責任壓力。

## Test Plan

- 確認所有 01 數據都以 65 份為分母。
- 確認開放題以 30 則非空回覆為基準，並扣除「無」或明顯無效內容後再更新主題命中。
- 檢查 02 的 5 個洞察是否仍各自有 `F/T` 證據支撐。
- 檢查文件是否避免把一對一信箱寫成「問卷已證明的最佳方案」。
- 檢查 03 是否維持假設型 Persona 語氣，且每張 Persona 有待驗證假設。
- 檢查新增風險是否被追蹤：平台差異化、分享者誘因、配對不對稱、回覆品質。

## Assumptions

- 更新方式採覆寫現有 01/02/03 文件。
- 新版 Excel 是正式資料來源。
- 目前不進入 User Stories 或 MVP 下一階段，直到 01/02/03 根據 65 份資料完成校準。
- 一對一匿名經驗交流信箱保留為主要產品假設，但研究文件需誠實呈現其仍待驗證。
