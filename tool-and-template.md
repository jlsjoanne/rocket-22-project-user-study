# 開發前使用者分析文件與模板

目前階段只是開發前的可能使用者分析，不需要建立過度複雜的研究文件。建議精簡成 5 份文件與 3 種模板即可。

## 一、建議產出文件

| 文件 | 用途 | 建議工具 |
|---|---|---|
| `01_問卷重點整理` | 放描述統計、重要比例、開放題重點 | Google Sheets |
| `02_需求洞察整理` | 把問卷與開放題收斂成 4-6 個需求主題 | FigJam / Notion |
| `03_假設型 Persona` | 只做 2 類：經驗尋求者、經驗分享者 | Notion |
| `04_User Stories` | 產出 6-8 則核心故事，附簡短證據與驗收條件 | Notion / Sheets |
| `05_MVP 建議與待驗證假設` | 排 Must / Should / Could，標註風險與後續要驗證的問題 | Notion |

## 二、建議使用模板

| 模板 | 用途 |
|---|---|
| Affinity Diagram | 整理需求洞察 |
| User Persona Template | 做 2 類假設型 Persona |
| User Story Template | 寫 User Stories 與簡單驗收條件 |

## 三、使用原則

- 不另外製作完整競品分析文件，只在 `05_MVP 建議與待驗證假設` 裡放一小段「參考平台觀察」即可。
- 不把 55 份問卷包裝成市場定論，只作為探索性需求線索。
- 文件重點是把問卷轉成可討論、可開發前評估的方向。
